# 照合と報告（進行役専用）

このファイルは進行役だけが読む。読者に見せない（採点基準を知った読者は回答を基準に寄せる）。SKILL.md 本体から分離してあるのは、読者側で Skill が誤発火してもこの内容がコンテキストに載らないようにするため。

## Step 3 — 照合の観点

- **中心主張・行動喚起**（Q1/Q2）が Step 0 のロックと一致するか
- **語り直し**（Q3）に中心主張と行動喚起が残っているか。細部が落ちるのは正常、**核が落ちていたら不合格**。逆に、意図していない要素が語り直しの主役になっていたら buried lede
- **決め手**（Q4b）が作者の意図した主役要素（図・数字・実物）と一致するか。決め手として**タイトルの文言しか挙がらない**スライドは、証拠が働いておらず「タイトルで言っているだけ」（故障クラス: 証拠が不発）
- **差分**（Q4a）で「重複」や空虚な回答が出たスライドは、構成上の冗長か孤児

## 合格基準（用途で変える）

| 用途 | 合格ライン |
|---|---|
| 登壇（口頭補完あり） | Q1/Q2 の一致＋語り直しに核が残存。スライド間の接続の飛躍は許容 |
| 読み資料 | 上記に加え、各スライドの差分（Q4a）が作者の意図した「伝えること」と概ね対応 |

## Step 4 — 報告フォーマット

```
## 伝達テスト結果: <合格 / ずれあり>

| 観点 | 意図 | 読者の再構成 | 判定 |
|---|---|---|---|
| 中心主張 | ... | ... | 一致/ずれ |
| 行動喚起 | ... | ... | 一致/欠落 |
| 語り直し（30秒） | （核＝中心主張+行動喚起） | ... | 核が残存/核が脱落 |

### スライド別の齟齬（あれば）
- スライドN: 意図した決め手「...」/ 読者の決め手「...」→ 故障クラス（証拠が不発 / 重複 / buried lede / 迷子 等）

### 修正提案
- どの層を直すべきか（構成・強調・メッセージ設計 / 文言）を明示する。
  ずれの原因はほぼ構成と強調にある。文言の表層修正で誤魔化さない。
```
