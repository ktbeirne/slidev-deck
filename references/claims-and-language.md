# 主張と文言の規律（Phase S / Phase C 共通）

絵コンテ（Phase S）と肉付け（Phase C）の両方から参照される。文言が確定するのは Phase C なので、最終執行点は C。

## 1. 事実・具体例の書き分け

- **社内・ドメイン固有の事実**（ツール名・事例・数値・体制）: 想像で生成しない。ブリーフの入力資料欄に列挙された資料からのみ引き、無ければ「依頼」に挙げる。
- **一般知識・Webで確認できる事実**（例: 公開規格の策定元や公開時期）: 確認伺いで止まらず**具体案として書き込む**（必要ならWebで裏取りする。曖昧な言い回しに逃げない）。ユーザは気に入らなければ差し替えを指示できる。

## 2. 増幅禁止

資料の事実を主張に使うとき、**資料の記載より強い断定に増幅しない**。同点は同点、「制約あり」は「制約あり」のまま保つ。「〜のみ」「全軸で最高」のような全称・排他の断定は、資料がそのまま支持する場合に限る。説得の方向への増幅は、聴衆が根拠表を見た瞬間に資料全体の信頼を壊す。

**チェックの操作化**（印象による「増幅なし」判定を認めない）:
1. 全称・排他・最上級の表現（「〜のみ」「唯一」「すべて」「全軸」「最高」「完全」等）を全文から機械的に抽出する。
2. 1件ずつ、資料の該当記述を引用して支持されるか照合する。

特定の語だけのパターン照合は、同型の増幅を語彙違いで見逃す。抽出は表現のカテゴリ（全称・排他・最上級）で行う。

## 3. 不利な事実の扱い

隠さない・盛らない（増幅禁止）は絶対条件。ただし**強調するかは目的に従うレトリック判断**であり義務ではない。提案・説得デッキで自案の弱点を視覚的に強調する必要はない — 正確に記載した上で、聴衆が必ず突く懸念にだけ応答を用意する。扱いの強さ（1枚割いて先回りするか、表の1行に留めて口頭説明で応えるか）は、聴衆の重視軸と口頭補完の有無から決める。

## 4. 文言規範

適用対象は**紙面に載る全文言** = ページタイトル・takeaway・主役・従属・注記（絵コンテのラフ行「伝えること／どう伝えるか」は設計メモなので対象外）。

- **ダッシュ（—/―/–）は引用以外禁止**。代替は「：」か句点分割。
- **ページ番号参照・次ページ誘導**は用途規則に従う: 報告・配布 = 許容、登壇 = 禁止。
- **入れ子引用の正規化**: 肉付けの実文言が外側「」で引用されている場合、文中の入れ子『』は紙面では「」に統一する（紙面には外側の引用符が出ないため、『』のまま転記すると理由のない二重鉤括弧が露出する）。書名等、本来『』であるべきものは除く。
- 発話・図ラベルを「」で引用している場合、紙面のコンテナ（吹き出し・図ラベル）内では外側「」を剥がす（コンテナ自体が引用装置）。

## 5. 文言スロップの指紋（負のリスト）

webライティング・ブログ記事の定型を紙面に持ち込まない。1文ごとには自然でも、組み合わさると「量産されたブログの導入」の型になり、資料の信頼を下げる。導入・まとめページで特に出やすい。

- 呼びかけ疑問文の導入（「〜を見かけませんか」「〜で困ったことはありませんか」）
- 共感の定型（「それ、あなただけじゃありません」）
- 所要時間つきの効能の約束（「これを読めば5分で〜できるようになります」）
- 配布資料での登壇口調（「今日の〜」）

現状や範囲は事実として率直に書く。トーンの柔らかさは文体で出し、定型レトリックで出さない。**Phase S の「どう伝えるか」に共感・呼びかけ等の方針が書かれていても、この負のリストが文言化の際に優先する**（S = 型を選ぶ、C = 型を崩さず文言化する、の二層）。

## 6. ノイズゲート

各要素（特に注記・出典・補足キャプション・従属）について「ゴール達成・読者の納得形成に**何をもって与するか**」を1行で言えるか点検し、言えない要素は削る。「事実として正しい」「あって損はない」は残す理由にならない。

例外の判定も機能で行う: 信頼性が判断材料になるデッキ（報告・決裁）では、出典・計測条件はゴールに与する側なので機械的に削らない。
