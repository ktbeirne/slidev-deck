# 作図構想の委託（Phase S 手順4）

## いつ使うか

導入・締め等の**型が無い、視覚で見せるページ**（図・実例）が対象。「どう伝えるか」を自前の形式カタログ（吹き出し／表／フロー…）から選ばない。構想を絵コンテ全体の流れ作業の1行として扱うと、注意の希釈と手持ち形式への引き寄せで「意味を運ばない図」（両端だけ描いて中間過程が無い等）が素通りする。既存規則は誤含意・ノイズを禁じる負の規制のみであり、**「図は新情報を運べ」という正の要求はこの委託の検収でだけ執行される**。隔離と1ページ集中の有効性は独立試行3/3で検証済み。

## 委託の入力契約

隔離した構想エージェントに渡すのは**「デッキの前提（聴衆・用途・トーン）＋当該ページの伝えること」だけ**。入力資料・他ページ・確立済み視覚語彙・既存実装は渡さない。出力は「読者が何を見れば理解できるか」の図の構想（種類・構成要素・ラベル・関係・配置）。

## 委託プロンプト（定型文。これ以外を書かない）

穴埋めは転記のみ。説明・例示・構造の示唆・「〜という構造を示せ」等の**自由記述を一切追加しない**。

> **理由**: プロンプトの自由著述は文脈保持者からの漏洩経路になる。実証例: 委託者が既存図の構造を「新情報の構造」として問題文に書き込み、構想が既存骨格に収束して委託が無意味化した。

報告には送信した委託プロンプト全文を含め、定型との一致をゲートで検品できるようにする。

```
<ブリーフのテーマ欄を転記>について解説するスライドを作成しています。以下のページに入れる解説用の図を考えてください。

- 聴衆: <ブリーフの聴衆欄を転記>
- 用途: <ブリーフの用途欄を転記>
- トーン: <ブリーフのトーン欄を転記>

## <ページタイトルを転記>
- 伝えること: <storyboard の伝えること欄を原文転記>

出力: 図の構想を言語で設計してください。図の種類、構成要素、各要素のラベル文言、要素間の関係（矢印・配置）を、実装者がそのまま作れる具体度で記述してください。

注意: ツールやファイルは使わないでください。この情報だけで設計してください。画像は生成しないでください。
```

## 検収基準（図だけテスト）

伝えることの**名詞（実体）と動詞（関係）を機械的に全行列挙した対応表**を作り、各行に対応する視覚要素を指させること。

- デッキ内で既出の要素は「既出のため図中不要」と理由を明示して除外してよい。**黙って行から落とさない** — 除外判断を表の上で見える化する。
- 読者が既に知っている光景の再現（例: チャットの依頼と回答の吹き出しだけ）は不合格として再委託する。

## 記録と責務分界

- 検収した構想は当該ページに `- 図の構想:` ブロックとして記録し、ゲートの確認対象に含める。
- ページ間の統一（確立済み視覚文法への翻訳）と空間予算への刈り込みは、採否の際に Phase S の実行者（構成作家）が行う。統一の対象は**語彙だけでなく、工程・サイクル等の順序と番号**を含む（別ページで確立済みの並びを、再掲時に別の順序・別の番号で回さない。委託は1ページ単位なので、ページ間整合は構造的に構成作家にしか見えない）。
- 実文言化と増幅・ノイズの規律は Phase C の責務。構想エージェントの出力は予算超過・軽い増幅を含みがちであり、**生成と規律を分離する**。

## 構成作家の権限（限定列挙）

受領した構想に対して構成作家ができるのは次の2つ**だけ**。

1. **差し戻し**: デッキ全体の流れとの不整合（他ページとの図の型・比喩の重複、確立済み視覚文法との衝突、物理的に紙面に収まらない等）がある場合、その**制約の事実だけ**を添えて構想エージェントに再設計させる。解決案・構造の指定はしない — 指定は委託プロンプト汚染と同じ漏洩経路。
2. **軽微な修正**: ラベルの用語をデッキの確立済み用語に揃える、表記正規化など、構造に触れない表層の統一のみ。

要素の削除・縮小・構造の変更はどちらの権限にも含まれない — 構想の中身を変えたければ差し戻して設計者にやらせる。検収（図だけテスト）を通った構想は、この2権限の適用を経て**そのまま** `- 図の構想:` ブロックになる。

> **理由**: 採否・刈り込みの裁量を構成作家に持たせると、負の規制を盾に提案を削って既存の形に回帰する勾配が働く（実証済み: 差し戻しページの再構想で委託提案がほぼ全棄却され、矢印ラベル追加だけに縮退した）。禁止の追記で縛るのではなく、権限自体を狭くする。
