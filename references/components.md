# 部品カタログ（Phase I）

実装前に必ずこの棚とデッキの style-notes.md を棚卸しし、該当部品があれば流用する。各部品の props・slots の詳細は `template/components/*.vue` の冒頭コメントが正。実レンダの見本は `template/type-specimen.md`（`slides-export/type-specimen/` に検分済みPNG）。

## ページ型部品（該当する型のページは必ずこれで実装する）

型の強制はテンプレートで行う。同じ構造を生HTMLで組み直さない。

| 部品 | 型 | 構成 |
|---|---|---|
| IntroToc | 解説・教育の導入 | takeaway（モチベーション1文、ページ側）＋「分かること」目次＋読後ゴール1行 |
| BigStatement | 定義・大メッセージ | display級の主文を中央配置（主文が takeaway 本体。別途 takeaway を置かない）＋補足＋注記 |
| RecapList | 解説・教育の締め | takeaway（自己チェックの促し、ページ側）＋要点再掲の番号リスト。**新規の主張を足さない** |
| FigurePage | 図解ページの標準骨格 | リード→視覚ブロック→下テキスト→注記。縦リズムを部品が固定。視覚ブロックの中身は自由（SVG図・表・グリッド）。Before/After は視覚ブロックに Compare を入れる。要素が多いページは `compact` |
| FullVisual | 図のみ型 | 全面アーキテクチャ図・全面比較表など、リード・下文なしで本文領域全体を視覚ブロック1つに使う。**条件: ページの主張はタイトルが担う（主張型タイトル）。題目型デッキでは「山場1枚の主張型格上げ」規定（Phase S）を使う** — 「主張が紙面に明文化されないページを作らない」規則との両立条件 |

## 要素部品（該当があれば使う）

| 部品 | 用途 |
|---|---|
| Badge | 色付きラベルの基礎プリミティブ（solid/soft/outline）。**ラベル・チップ・ピル状の造形はまず Badge で表現できないか検討**し、新しいCSSクラスを作らない |
| Compare | 対比2パネル＋中央矢印（Before/After） |
| FlowSteps | 手順の横フロー図（箱＋矢印＋丸番号＋任意アイコン） |
| KeyNumber | 大数字コールアウト（結論を支える数字1つ） |
| LeadBox | リード囲み: リードタイトル＋sub文を色面ブロックで囲むリード部の変種。FigurePage の #lead 内でも単独でも使える。takeaway の代替なので1ページ1つまで |
| ChatExample + ChatBubble | 会話例（チャット再現）。題材依存（AI・対話UI系）のため必須型ではない |

## 運用規則

- **自作は該当部品が無い場合のみ**。部品化された造形は品質と一貫性が既に担保されている — 自作は都度その担保を失う。
- **ページ型への昇格基準は「デッキ構造由来で毎回出るか」**。実例の有無だけでは昇格しない（題材依存の型は要素部品に置く）。
- 同じ視覚パターンが2回目に出たら初回の実装を再利用する（都度の別実装は見た目の一貫性と保守性を壊す）。
- アイコンは既成セット（iconify 経由・MDI、`<mdi-xxx />` 記法）から選ぶ。自作SVGパスを描かない。依存 `@iconify-json/mdi` は template に導入済み。
